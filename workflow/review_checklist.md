# 架构 / 代码 / 数据库 / 上线 Review 清单

适用于你做架构评审、代码评审、接口评审、SQL 评审和上线前检查时快速过一遍关键问题。

## 1. 需求与目标 Review

先确认：
- 这个改动到底要解决什么问题
- 目标是否清晰、可验证
- 是否有明确成功标准
- 是修复、优化、重构还是新增功能
- 是否存在被误解的需求边界

## 2. 架构方案 Review

### 2.1 方案层
- 是否真的解决了核心问题
- 是否过度设计
- 是否有更简单的方案
- 当前方案的 trade-off 是否明确
- 是否考虑未来 3~12 个月演进

### 2.2 模块边界
- 模块职责是否清晰
- 依赖方向是否合理
- 是否引入不必要耦合
- 是否存在隐式共享状态
- 是否容易扩展和替换

### 2.3 调用链与数据流
- 调用链是否清晰
- 错误如何传递
- 日志和监控打在哪里
- 是否存在状态不一致风险
- 是否存在链路过长问题

## 3. 代码 Review

### 3.1 可读性
- 命名是否准确
- 结构是否清晰
- 是否容易看懂主流程
- 是否存在过度抽象或过度嵌套
- 是否有无意义注释或缺少关键注释

### 3.2 职责与边界
- 函数职责是否单一
- 业务逻辑是否放在正确层级
- 是否把复杂逻辑堆在 handler / controller / main 中
- 是否存在 utils 泛化承载业务逻辑问题

### 3.3 错误处理
- 是否吞错
- 错误是否带上下文
- 对外错误与内部错误是否分层
- 是否方便日志定位

### 3.4 并发与资源
- goroutine 生命周期是否可控
- 是否正确处理 context
- 是否有资源泄漏风险
- 是否有竞态条件
- channel 是否使用合理

### 3.5 可测试性
- 关键逻辑是否容易测试
- 是否需要补单元测试或集成测试
- 核心分支是否可验证
- 修复 bug 是否补了回归测试

## 4. 前端 Review

- 页面是否覆盖加载态、空态、错误态、权限态
- 组件边界是否合理
- 状态是否单一可信
- 是否有重复请求、重复渲染或重复提交问题
- API 调用异常是否有统一处理
- 表单校验与错误提示是否清晰
- 权限控制是否到位
- 交互体验是否一致

## 5. 接口 Review

- 路径和语义是否清晰
- 请求响应结构是否稳定
- 字段命名是否准确
- 是否明确分页、过滤、排序、鉴权
- 错误码是否清晰稳定
- 是否兼容旧版本调用方
- 是否考虑幂等与重复提交问题

## 6. 数据库 / SQL Review

### 6.1 表结构
- 是否准确表达业务语义
- 是否有必要约束
- 是否容易出现脏数据
- 是否有合理默认值和唯一性规则

### 6.2 SQL
- 是否存在 `select *`
- where 条件和 join 条件是否清晰
- 是否会触发全表扫描
- 是否存在 N+1 问题
- 分页方式是否合理
- 更新和删除是否有明确条件

### 6.3 索引
- 索引是否服务真实查询路径
- 是否存在冗余索引
- 联合索引顺序是否合理
- 是否影响高频写入性能

### 6.4 事务与一致性
- 事务边界是否合理
- 是否存在长事务风险
- 是否有并发覆盖或重复写入风险
- 是否需要乐观锁、悲观锁或幂等控制

## 7. 安全 Review

至少检查：
- 是否做了鉴权与权限控制
- 是否有输入校验
- 是否存在 SQL 注入 / XSS / CSRF 风险
- 是否暴露敏感信息
- 日志是否泄露敏感数据
- 文件上传或第三方回调是否做安全校验
- 是否需要限流、防刷、防重

## 8. 可观测性 Review

- 关键链路是否有日志
- 错误日志是否有上下文
- 是否有 trace_id / request_id
- 是否有关键指标监控
- 是否有超时、错误率、依赖成功率监控
- 故障发生时是否能快速定位

## 9. 性能 Review

- 是否识别了当前真实瓶颈
- 是否存在明显低效查询或重复请求
- 是否会造成热点资源竞争
- 是否需要缓存、批处理、异步化或限流
- 是否属于过早优化

## 10. 重构 Review

- 这次重构是否真的必要
- 是否目标明确
- 是否采用渐进式方案
- 是否夹带无关改动
- 是否保留回滚路径
- 是否能验证新旧逻辑一致性

## 11. 上线 Review

上线前至少确认：
- 功能主路径已验证
- 异常分支已验证
- 配置已检查
- 数据库脚本已审核
- 外部依赖已确认
- 监控与告警已准备
- 回滚方案明确
- 高风险改动是否灰度

## 12. Review 结论模板

每次评审后，建议输出：

### 结论
- 可以合并 / 可以上线 / 需要修改后再评审

### 主要问题
- 问题 1
- 问题 2
- 问题 3

### 风险等级
- 高 / 中 / 低

### 必改项
- 必须修复的问题

### 建议项
- 可优化但不阻塞当前推进的问题

### 验证建议
- 建议补哪些测试、日志、监控或灰度验证
