# Publish Checklist

用于 GitHub 公开发布前的最后检查。

---

## 1. 内容检查

发布前确认：

- [ ] 不包含私有项目实例目录
- [ ] 不包含企业内部规范
- [ ] 不包含敏感业务结构、内部接口或内部流程
- [ ] 不包含明显个人化或不可公开的表述
- [ ] 不包含路径、文件名或内容中的隐私信息

---

## 2. 结构检查

确认这些目录和文件存在且可读：

- [ ] `README.md`
- [ ] `LICENSE`
- [ ] `SKILL_MANIFEST.md`
- [ ] `CHANGELOG.md`
- [ ] `CONTRIBUTING.md`
- [ ] `core/`
- [ ] `adapters/`
- [ ] `rules/`
- [ ] `workflow/`
- [ ] `examples/`
- [ ] `packs_rendered/`

---

## 3. 命名检查

- [ ] 仓库名清晰
- [ ] Skill 名称清晰
- [ ] 文件命名统一
- [ ] 不再使用私有项目名或个人内部命名

---

## 4. README 检查

- [ ] 首页说明清晰
- [ ] Quick start 清晰
- [ ] 目标用户明确
- [ ] 使用方式明确
- [ ] 公共范围与边界说明明确

---

## 5. License 检查

- [ ] 已确认使用 MIT
- [ ] `LICENSE` 文件存在
- [ ] 如有需要，补充 copyright

---

## 6. 可用性检查

- [ ] Claude 成品版可直接复制使用
- [ ] Cursor 成品版可直接复制使用
- [ ] 专项成品版命名清楚
- [ ] 示例模板可直接复制使用

---

## 7. 发布后动作

建议发布后立刻补充：

- [ ] 仓库 description
- [ ] Topics / tags
- [ ] 首个 release
- [ ] 版本号标记
- [ ] 后续 roadmap

---

## 8. 一句话发布标准

如果这套内容让陌生用户打开仓库后，能在 3 分钟内理解：
- 这是什么
- 适合谁
- 怎么用
- 从哪里开始

那么它就已经具备公开发布的基本条件。
