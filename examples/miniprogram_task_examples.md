# 小程序任务示例

适用于微信小程序、支付宝小程序、抖音小程序等场景。目标是让 AI 在小程序场景下一开始就考虑生命周期、登录态、分享、支付、审核和发布，而不是只按 H5 思路回答。

---

## 1. 设计一个小程序页面

### 示例提示词
请你按小程序专项标准，帮我设计一个页面或功能。

需求：
- 

页面类型：
- 列表页 / 详情页 / 表单页 / 支付页 / 活动页

约束：
- 需要考虑页面生命周期
- 需要考虑登录态与 session
- 需要考虑弱网和切后台恢复
- 如涉及分享 / 支付 / 订阅消息，也要纳入设计

请输出：
1. 页面结构建议
2. 生命周期和状态设计
3. 登录与接口协作建议
4. 分享 / 支付 / 订阅消息设计（如适用）
5. 性能与审核注意事项
6. 风险点
